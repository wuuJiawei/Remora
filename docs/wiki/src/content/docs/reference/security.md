---
title: 安全设置
description: 了解 Remora 的安全机制和配置选项。
---

Remora 采用本地优先的安全策略，保护您的凭据和连接安全。

## 凭据存储

### 本地配置存储

Remora 将密码与配置保存在本机 `~/.config/remora` JSON 文件中：

- 已保存密码以明文形式写入本地配置目录
- 首次连接时询问是否保存
- 导出与复制明文密码仍需要用户主动确认

### 导出凭据

导出主机配置时，可以选择是否包含已保存的密码。包含密码的导出文件会以明文存储，请妥善保管。

## SSH 密钥

### 支持的密钥格式

Remora 支持以下 SSH 密钥格式：

- RSA (2048/4096 位)
- ED25519
- ECDSA (256/384/521 位)

### 密钥位置

密钥文件通常存放在 `~/.ssh/` 目录：

```
~/.ssh/id_rsa
~/.ssh/id_ed25519
~/.ssh/id_ecdsa
```

### 密钥权限

确保私钥文件权限正确：

```bash
chmod 600 ~/.ssh/id_ed25519
```

## 主机指纹

首次连接新主机时，Remora 会显示主机指纹供您确认。您可以选择接受或拒绝该主机密钥。
