---
title: 传输管理
description: 管理 SFTP 文件传输任务。
---

Remora 提供完整的文件传输管理功能，包括队列、暂停、续传等。

## 传输队列

所有 SFTP 传输操作都会进入队列统一管理。

### 查看队列

点击侧边栏底部的 **传输** 按钮，或按 `Cmd+Shift+F` 打开传输面板。

### 队列状态

| 状态 | 说明 |
|------|------|
| 等待中 | 等待传输 |
| 传输中 | 正在传输 |
| 已暂停 | 暂停传输 |
| 已完成 | 传输成功 |
| 失败 | 传输失败 |

## 传输操作

### 暂停与恢复

- **暂停**：点击暂停按钮暂停当前传输
- **恢复**：点击恢复按钮继续传输
- 暂停的传输可以随时恢复

### 取消传输

- 点击取消按钮终止传输
- 已传输的部分文件会被保留

### 重试失败传输

- 点击重试按钮重新传输失败的文件
- 支持断点续传（服务器支持时）

## 传输设置

在 **设置 > 文件管理器 > 传输** 中配置：

### 并发传输数

设置同时进行的最大传输任务数（默认：3）。

### 传输缓冲

设置文件传输缓冲大小，影响传输速度和内存占用。

### 覆盖策略

选择文件已存在时的处理方式：

- **询问**：每次询问用户
- **覆盖**：直接覆盖
- **跳过**：跳过该文件
- **重命名**：自动重命名为新名字

### 保留时间戳

启用后，上传/下载文件会保留原始文件的时间戳。

## 断点续传

Remora 支持断点续传，传输中断后可从中断处继续。

### 支持条件

- 服务器 SFTP 服务支持断点续传
- 传输设置中启用 **恢复支持**

### 使用方式

传输失败后，点击 **重试** 按钮即可从断点继续。
